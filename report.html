1. 执行摘要

问题成立
供给损耗主要集中在首发后的持续经营阶段；曝光集中度高于价值集中度。
推进方式
固定 Top N 历史回测支持进入首轮灰度；不支持直接全量切换。
读数原则
是否扩量由线上主指标与护栏决定，离线结论只用于判断是否值得试。

3. 方法与口径

01
供给漏斗诊断：先找损耗位置。
02
资源错配识别：比较曝光与价值集中度。
03
关键人群分层：识别高曝光低变现与高潜低激励。
04
规则设计：按准入、排序、复核组织首轮灰度。
05
固定 Top N 回测：控制规模一致后再比较。
06
实验设计：把主指标、护栏、SRM 与回滚条件写清楚。

首轮灰度评分公式

新优先级分数 = 0.60 × 原规则分位 + 0.20 × 留存分位 + 0.15 × 变现效率分位（缩尾后） + 0.05 × 低资源占用分位

阈值与分位口径

  • 高曝光低变现：曝光分位 ≥ 0.60，效率分位 ≤ 0.52。
  • 高潜低激励：留存分位 ≥ 0.65，效率分位 ≥ 0.65，资源占用分位 ≤ 0.68。
  • 高价值稳定供给：留存分位 ≥ 0.72，效率分位 ≥ 0.72。
  • 低价值供给：曝光分位 ≤ 0.25，效率分位 ≤ 0.35。

5. 实验与风控

项目建议
随机化单位creator 级分层随机实验
实验动作对高潜低激励池做增配；对高曝光低变现池做降配 / 复核
主指标30 日留存率；单位激励收入
护栏指标人均现金激励；供给流失率；投诉率；作弊率；内容安全
建议分层内容垂类；历史曝光档位；创作者体量；是否已开通变现
主读数原则以 intent-to-treat 为主，treatment-on-the-treated 只作辅助
扩量条件主指标不为负，单位激励收入保持为正，且关键护栏无明显恶化
暂停或回滚连续 3 天出现显著 SRM / 分流漂移，或投诉/作弊/安全明显恶化，或单位激励收入转负
风险控制方式
历史回测不是因果识别正式扩量前采用分层随机实验或 switchback 验证。
SRM / 分流漂移按日监控 creator 数、曝光和预算分配；显著偏离时暂停读数。
推荐外溢优先做池级隔离、creator 级锁定，并单独监控外溢敏感指标。
观察窗截断短窗看诊断指标，决策以 30 日或更长窗口为准。
新鲜感效应比较 week 1 与 week 4 效应，并在扩量后保留 holdout。
跨池迁移污染锁定 assignment 窗口，以 intent-to-treat 作为主读数。
回滚条件：连续 3 天出现显著 SRM / 分流漂移；单位激励收入转负且没有恢复迹象；投诉率、作弊率、内容安全明显恶化；week 1 抬升但 week 4 回落，且 holdout 不支持继续扩量。
