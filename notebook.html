分析过程

按经营判断顺序整理分析链路

这一页只保留做决策会用到的口径、步骤和代码片段，不写无关的“过程说明语句”。顺序是：定义问题、定位损耗、识别错配、构造分层、固定 Top N 回测、准备实验监控。

分析顺序

0. 目标定义

判断资源是否持续流向更可能形成稳定供给和单位资源产出的创作者。

1. 供给诊断

识别损耗阶段，明确资源动作作用在哪个经营环节。

2. 错配识别

比较曝光集中度与价值集中度，识别沉没资源与高潜低配。

3. 规则设计

规则由准入、排序和复核构成，同时控制迁移成本与承接压力。

4. 固定 Top N 回测

只在同规模下比较主指标，避免通过扩大样本池制造表面提升。

5. 实验准备

离线结论只决定是否值得试；是否扩量，要看线上主指标和护栏。

核心口径

创作者主表

creator_id first_publish_date active_weeks_30d retention_30d monetization_open_flag revenue_30d impression_30d cash_incentive_30d rpm_30d = revenue_30d / impression_30d * 1000 unit_incentive_revenue = revenue_30d / cash_incentive_30d

资源配置判断项

持续供给：retention_30d, active_weeks_30d 单位资源产出：rpm_30d, unit_incentive_revenue 供给质量：monetization_open_flag, 近期经营信号 风险约束：safe_flag, cheat_flag, complaint_flag

步骤与代码片段

步骤 1：供给漏斗

先找损耗位置，再决定资源动作作用在哪个阶段。

WITH base AS ( SELECT creator_id, CASE WHEN first_publish_date IS NOT NULL THEN 1 ELSE 0 END AS published, CASE WHEN active_weeks_30d >= 2 THEN 1 ELSE 0 END AS active_2w, CASE WHEN active_weeks_30d >= 4 THEN 1 ELSE 0 END AS active_4w, monetization_open_flag FROM creator_profile ) SELECT SUM(published) / COUNT(*) AS publish_rate, SUM(active_4w) / COUNT(*) AS stable_operation_rate, SUM(CASE WHEN active_4w = 1 AND monetization_open_flag = 1 THEN 1 ELSE 0 END) / COUNT(*) AS stable_monetized_rate FROM base;

步骤 2：资源错配

用曝光集中度和价值集中度做第一层判断。

SELECT creator_id, impression_30d, revenue_30d, SUM(impression_30d) OVER(ORDER BY impression_30d DESC) / SUM(impression_30d) OVER() AS cum_impression_share, SUM(revenue_30d) OVER(ORDER BY impression_30d DESC) / SUM(revenue_30d) OVER() AS cum_revenue_share FROM creator_profile;

步骤 3：关键人群分层

分层的目的不是描述人群，而是服务差异化动作。

CASE WHEN impression_pct >= 0.8 AND rpm_30d < p30_rpm THEN '高曝光低变现' WHEN retention_30d >= p75_retention AND unit_incentive_revenue >= p75_unit_rev AND impression_pct < 0.5 THEN '高潜低激励' WHEN retention_30d >= p75_retention AND unit_incentive_revenue >= p50_unit_rev THEN '高价值稳定供给' WHEN retention_30d < p25_retention AND unit_incentive_revenue < p25_unit_rev THEN '低价值供给' ELSE '一般供给' END AS creator_segment

步骤 4：新规则打分

旧规则保留主体，新信号只做首轮修正，目的是控制迁移速度。

priority_score = 0.60 * old_rule_percentile + 0.20 * retention_percentile + 0.15 * efficiency_percentile + 0.05 * low_resource_occupancy_percentile

步骤 5：固定 Top N 回测

比较对象必须在同规模下产生，避免“扩池”带来的虚假改善。

WITH ranked AS ( SELECT *, ROW_NUMBER() OVER (ORDER BY old_rule_score DESC) AS rn_old, ROW_NUMBER() OVER (ORDER BY priority_score DESC) AS rn_new FROM creator_scored ) SELECT AVG(CASE WHEN rn_old <= 4200 THEN retention_30d END) AS old_retention, AVG(CASE WHEN rn_new <= 4200 THEN retention_30d END) AS new_retention, AVG(CASE WHEN rn_old <= 4200 THEN unit_incentive_revenue END) AS old_unit_rev, AVG(CASE WHEN rn_new <= 4200 THEN unit_incentive_revenue END) AS new_unit_rev FROM ranked;

步骤 6：实验监控

主指标和护栏同时准备，避免离线结果一好看就直接扩量。

核心监控： 1) SRM：样本量、曝光、预算分配是否偏离 2) 主指标：30日留存率、单位激励收入 3) 护栏：人均现金激励、供给流失率、投诉率、作弊率、安全 4) 观察窗：week1、week2、week4 与 holdout 对照

读数纪律

约束要求
比较口径所有策略比较都在固定 Top N 下完成。
解释方式离线 uplift 只解释方向，不解释真实线上因果增量。
扩量规则主指标为正且关键护栏稳定后，再按预算档位扩量。
回滚条件连续 3 天 SRM 显著偏移，或单位激励收入转负，或风险侧恶化。
样本说明：数据为合成样本；离线结果仅用于判断是否进入实验，不替代线上因果识别。
