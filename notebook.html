分析过程

经营判断顺序

分析顺序与代码片段一一对应。

0. 目标定义

聚焦资源配置效率判断。

1. 供给诊断

识别损耗阶段，明确动作落点。

2. 错配识别

比较曝光集中度与价值集中度，识别沉没资源与高潜低配。

3. 规则设计

规则由准入、排序和复核三部分组成，并兼顾迁移成本和运营承接。

4. 固定 Top N 回测

只在同规模下比较主指标，避免通过扩大样本池制造表面提升。

5. 实验准备

离线证据只决定是否值得试；是否扩量，要看线上主指标和护栏。

0. 分析目标

核心问题：当流量和现金激励都有限时，平台是否把资源给到了更值得长期扶持的创作者？

import pandas as pd import numpy as np creator = pd.read_csv('../data/creator_profile.csv', parse_dates=['join_date']) content = pd.read_csv('../data/content_performance.csv', parse_dates=['publish_date']) weekly = pd.read_csv('../data/creator_weekly_activity.csv') incentive = pd.read_csv('../data/creator_incentive.csv') print('creator rows:', len(creator)) print('content rows:', len(content)) print('weekly rows:', len(weekly)) print('incentive rows:', len(incentive))
当前页面聚焦资源重配方向与灰度条件。
1. Funnel

供给漏斗

如果主要损耗发生在注册到首发之前，动作就偏拉新和首发激活；如果发生在首发之后，动作重点就应转向持续经营。

funnel = pd.DataFrame({ 'stage': ['注册创作者','完成首发','连续活跃≥2周','连续活跃≥4周','已开通变现','稳定经营且已开通变现'], 'creator_cnt': [ len(creator), int(creator['has_first_publish'].sum()), int(((creator['has_first_publish'] == 1) & (creator['active_weeks'] >= 2)).sum()), int(((creator['has_first_publish'] == 1) & (creator['active_weeks'] >= 4)).sum()), int(((creator['has_first_publish'] == 1) & (creator['active_weeks'] >= 4) & (creator['monetization_opened'] == 1)).sum()), int(((creator['has_first_publish'] == 1) & (creator['active_weeks'] >= 4) & (creator['monetization_opened'] == 1) & (creator['stable_updater'] == 1)).sum()) ] }) funnel['share_of_registered'] = funnel['creator_cnt'] / funnel.loc[0, 'creator_cnt'] funnel['step_conversion'] = funnel['creator_cnt'] / funnel['creator_cnt'].shift(1)
业务判断主要损耗集中在首发后的持续经营阶段，因此资源动作应优先提高已启动创作者进入稳定经营状态的概率。
2. Cohort

cohort 稳定性

排除样本质量突变和短观察窗干扰。

cohort = ( creator.assign(join_month=creator['join_date'].dt.to_period('M').astype(str)) .groupby('join_month') .agg( creators=('creator_id', 'count'), publish_rate=('has_first_publish', 'mean'), retention_30d=('retention_30d_rate', 'mean'), monetization_open_rate=('monetization_opened', 'mean') ) .reset_index() )
业务判断各 cohort 整体平稳，问题更可能来自资源分配逻辑。最新 cohort 样本较小，仅作预警信号。
3. Misallocation

识别资源错配

用曝光集中度与收入集中度的差异判断资源是否仍沿历史曝光惯性分配，并用分群识别首轮动作人群。

pareto = ( creator.sort_values('total_exposure', ascending=False) .assign( creator_share=lambda x: np.arange(1, len(x)+1) / len(x), cum_exposure_share=lambda x: x['total_exposure'].cumsum() / x['total_exposure'].sum(), cum_revenue_share=lambda x: x['total_revenue'].cumsum() / x['total_revenue'].sum() ) ) top20 = creator.nlargest(int(len(creator) * 0.2), 'total_exposure') exposure_share = top20['total_exposure'].sum() / creator['total_exposure'].sum() revenue_share = top20['total_revenue'].sum() / creator['total_revenue'].sum()
业务判断Top 20% 创作者拿走 42.9% 曝光，只贡献 30.2% 收入，说明当前资源集中度高于价值集中度。
4. Rule design

定义首轮灰度口径

首轮规则兼顾迁移可控、主指标方向和运营承接。

creator['old_rule_pct'] = creator['old_priority_score'].rank(pct=True) creator['retention_pct'] = creator['retention_30d_rate'].rank(pct=True) creator['efficiency_pct'] = creator['revenue_per_k_exposure_clip'].rank(pct=True) creator['low_resource_pct'] = (-creator['resource_occupancy_pct']).rank(pct=True) creator['new_priority_score'] = ( 0.60 * creator['old_rule_pct'] + 0.20 * creator['retention_pct'] + 0.15 * creator['efficiency_pct'] + 0.05 * creator['low_resource_pct'] )
业务判断旧规则权重保留较高，用于控制名单迁移速度；留存进入排序主逻辑，把长期供给质量显式纳入分发。
5. Same-scale backtest

固定 Top N 做回测

所有比较都在固定 Top N 下完成，避免通过扩大样本池制造表面提升。

TOP_N = 4200 old_sel = creator.nlargest(TOP_N, 'old_priority_score').copy() new_sel = creator.nlargest(TOP_N, 'new_priority_score').copy() summary = pd.DataFrame({ 'metric': ['retention_30d_rate', 'revenue_per_incentive', 'cash_incentive_per_creator'], 'old_rule': [old_sel['retention_30d_rate'].mean(), old_sel['revenue_per_incentive'].mean(), old_sel['cash_incentive'].mean()], 'new_rule': [new_sel['retention_30d_rate'].mean(), new_sel['revenue_per_incentive'].mean(), new_sel['cash_incentive'].mean()] })
业务判断30 日留存和单位激励收入同步改善，新旧名单重合率约 72.2%，调整幅度可控。
6. Launch readiness

补上实验与风控

离线证据用于判断是否进入灰度；是否扩量，需要主指标、护栏、暂停条件和风控动作同时成立。

experiment_card = { 'randomization_unit': 'creator 级分层随机实验', 'primary_metrics': ['30 日留存率', '单位激励收入'], 'guardrails': ['人均现金激励', '供给流失率', '投诉率', '作弊率', '内容安全'], 'expand_if': '主指标不为负，单位激励收入为正，关键护栏无明显恶化', 'rollback_if': '连续 3 天出现显著 SRM / 分流漂移，或投诉 / 作弊 / 安全明显恶化' }
