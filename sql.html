SQL 方案

把经营判断落到可复核的 SQL

SQL 按事实层、指标层、策略层与实验监控组织，覆盖供给漏斗、cohort、关键人群、固定 Top N 回测、名单迁移和灰度监控。

1. Funnel

供给漏斗

确认问题发生在注册、首发或持续经营阶段。

WITH creator_base AS ( SELECT creator_id, join_date, has_first_publish, active_weeks, monetization_opened, stable_updater FROM creator_profile ), funnel AS ( SELECT '注册创作者' AS stage, COUNT(*) AS creator_cnt FROM creator_base UNION ALL SELECT '完成首发', COUNT(*) FROM creator_base WHERE has_first_publish = 1 UNION ALL SELECT '连续活跃≥2周', COUNT(*) FROM creator_base WHERE has_first_publish = 1 AND active_weeks >= 2 UNION ALL SELECT '连续活跃≥4周', COUNT(*) FROM creator_base WHERE has_first_publish = 1 AND active_weeks >= 4 UNION ALL SELECT '已开通变现', COUNT(*) FROM creator_base WHERE has_first_publish = 1 AND active_weeks >= 4 AND monetization_opened = 1 UNION ALL SELECT '稳定经营且已开通变现', COUNT(*) FROM creator_base WHERE has_first_publish = 1 AND active_weeks >= 4 AND monetization_opened = 1 AND stable_updater = 1 ) SELECT * FROM funnel;
2. Cohort

Cohort 稳定性

避免把短观察窗或样本小波动误读成资源策略问题。

SELECT DATE_FORMAT(join_date, '%Y-%m') AS join_month, COUNT(*) AS creators, AVG(has_first_publish) AS publish_rate, AVG(retention_30d_rate) AS retention_30d, AVG(monetization_opened) AS monetization_open_rate FROM creator_profile GROUP BY 1 ORDER BY 1;
3. Segmentation

关键人群分层

给首轮动作提供名单来源。

WITH scored AS ( SELECT creator_id, exposure_share_pct, revenue_per_k_exposure, retention_30d_rate, resource_occupancy_pct, CASE WHEN exposure_share_pct >= 0.80 AND revenue_per_k_exposure < 50 THEN '高曝光低变现' WHEN retention_30d_rate >= 0.60 AND revenue_per_k_exposure >= 90 AND resource_occupancy_pct < 0.40 THEN '高潜低激励' WHEN retention_30d_rate >= 0.60 THEN '高价值稳定供给' WHEN retention_30d_rate < 0.35 THEN '低价值供给' ELSE '一般供给' END AS segment FROM creator_profile ) SELECT segment, COUNT(*) AS creators FROM scored GROUP BY 1 ORDER BY creators DESC;
4. Rule scoring

首轮灰度评分

首轮口径核心是：控制迁移速度，让留存显式进入排序，同时把效率和低占用作为修正项。

WITH ranked AS ( SELECT creator_id, PERCENT_RANK() OVER (ORDER BY old_priority_score) AS old_rule_pct, PERCENT_RANK() OVER (ORDER BY retention_30d_rate) AS retention_pct, PERCENT_RANK() OVER (ORDER BY revenue_per_k_exposure_clip) AS efficiency_pct, PERCENT_RANK() OVER (ORDER BY -resource_occupancy_pct) AS low_resource_pct FROM creator_profile ) SELECT creator_id, 0.60 * old_rule_pct + 0.20 * retention_pct + 0.15 * efficiency_pct + 0.05 * low_resource_pct AS new_priority_score FROM ranked;
5. Same-scale backtest

固定 Top N 回测

所有比较必须在同规模下完成，否则容易通过扩大样本池制造表面提升。

WITH old_sel AS ( SELECT creator_id FROM creator_profile ORDER BY old_priority_score DESC LIMIT 4200 ), new_sel AS ( SELECT creator_id FROM scoring_result ORDER BY new_priority_score DESC LIMIT 4200 ) SELECT 'old_rule' AS rule_type, AVG(retention_30d_rate) AS retention_30d, AVG(revenue_per_incentive) AS revenue_per_incentive, AVG(cash_incentive) AS cash_incentive FROM creator_profile WHERE creator_id IN (SELECT creator_id FROM old_sel) UNION ALL SELECT 'new_rule', AVG(retention_30d_rate), AVG(revenue_per_incentive), AVG(cash_incentive) FROM creator_profile WHERE creator_id IN (SELECT creator_id FROM new_sel);
6. Transition and monitoring

迁移结构与实验监控

输出名单迁移结构，并补充实验监控查询。

-- 名单迁移 SELECT CASE WHEN o.creator_id IS NOT NULL AND n.creator_id IS NULL THEN '旧名单移出' WHEN o.creator_id IS NULL AND n.creator_id IS NOT NULL THEN '新名单补入' END AS direction, c.segment, COUNT(*) AS creators, AVG(c.retention_30d_rate) AS retention_30d, AVG(c.revenue_per_incentive) AS revenue_per_incentive FROM creator_segment c LEFT JOIN old_sel o ON c.creator_id = o.creator_id LEFT JOIN new_sel n ON c.creator_id = n.creator_id WHERE (o.creator_id IS NOT NULL AND n.creator_id IS NULL) OR (o.creator_id IS NULL AND n.creator_id IS NOT NULL) GROUP BY 1,2; -- 日级 SRM / 分流漂移监控 SELECT dt, exp_group, COUNT(DISTINCT creator_id) AS creators, SUM(exposure) AS exposure, SUM(budget_cost) AS budget_cost FROM experiment_daily_fact GROUP BY 1,2 ORDER BY 1,2;
