SQL 方案

围绕经营判断整理 SQL，而不是按表名堆脚本

这一页只保留会真正用于资源重配判断的查询：供给漏斗、cohort、价值排序、人群分层、固定 Top N 回测、名单迁移、SRM 与护栏监控。

1. 供给漏斗

WITH base AS ( SELECT creator_id, CASE WHEN first_publish_date IS NOT NULL THEN 1 ELSE 0 END AS is_published, CASE WHEN active_weeks_30d >= 2 THEN 1 ELSE 0 END AS active_2w, CASE WHEN active_weeks_30d >= 4 THEN 1 ELSE 0 END AS active_4w, monetization_open_flag FROM creator_profile ) SELECT '注册创作者' AS stage, COUNT(*) AS creator_cnt FROM base UNION ALL SELECT '完成首发', SUM(is_published) FROM base UNION ALL SELECT '连续活跃≥2周', SUM(active_2w) FROM base UNION ALL SELECT '连续活跃≥4周', SUM(active_4w) FROM base UNION ALL SELECT '稳定经营且已开通变现', SUM(CASE WHEN active_4w = 1 AND monetization_open_flag = 1 THEN 1 ELSE 0 END) FROM base;

2. Cohort 留存与变现开通

SELECT DATE_TRUNC('month', first_publish_date) AS cohort_month, AVG(CASE WHEN retention_30d = 1 THEN 1.0 ELSE 0 END) AS retention_30d_rate, AVG(CASE WHEN monetization_open_flag = 1 THEN 1.0 ELSE 0 END) AS monetization_open_rate FROM creator_profile WHERE first_publish_date IS NOT NULL GROUP BY 1 ORDER BY 1;

3. 价值排序与错配识别

WITH ranked AS ( SELECT creator_id, impression_30d, revenue_30d, NTILE(100) OVER (ORDER BY impression_30d DESC) AS impression_pct_desc, SUM(impression_30d) OVER (ORDER BY impression_30d DESC) / SUM(impression_30d) OVER () AS cum_impression_share, SUM(revenue_30d) OVER (ORDER BY impression_30d DESC) / SUM(revenue_30d) OVER () AS cum_revenue_share FROM creator_profile ) SELECT * FROM ranked WHERE impression_pct_desc <= 20;

4. 分层与新规则打分

WITH feature_base AS ( SELECT creator_id, old_rule_percentile, retention_percentile, efficiency_percentile, low_resource_occupancy_percentile, CASE WHEN impression_percentile >= 80 AND rpm_30d < p30_rpm THEN '高曝光低变现' WHEN retention_30d >= p75_retention AND unit_incentive_revenue >= p75_unit_rev AND impression_percentile < 50 THEN '高潜低激励' WHEN retention_30d >= p75_retention AND unit_incentive_revenue >= p50_unit_rev THEN '高价值稳定供给' WHEN retention_30d < p25_retention AND unit_incentive_revenue < p25_unit_rev THEN '低价值供给' ELSE '一般供给' END AS creator_segment FROM creator_feature_table ) SELECT *, 0.60 * old_rule_percentile + 0.20 * retention_percentile + 0.15 * efficiency_percentile + 0.05 * low_resource_occupancy_percentile AS priority_score FROM feature_base;

5. 固定 Top N 回测

WITH ranked AS ( SELECT *, ROW_NUMBER() OVER (ORDER BY old_rule_score DESC) AS rn_old, ROW_NUMBER() OVER (ORDER BY priority_score DESC) AS rn_new FROM creator_scored ) SELECT AVG(CASE WHEN rn_old <= 4200 THEN retention_30d END) AS old_retention, AVG(CASE WHEN rn_new <= 4200 THEN retention_30d END) AS new_retention, AVG(CASE WHEN rn_old <= 4200 THEN unit_incentive_revenue END) AS old_unit_incentive_revenue, AVG(CASE WHEN rn_new <= 4200 THEN unit_incentive_revenue END) AS new_unit_incentive_revenue, AVG(CASE WHEN rn_old <= 4200 THEN cash_incentive_30d END) AS old_avg_cash_incentive, AVG(CASE WHEN rn_new <= 4200 THEN cash_incentive_30d END) AS new_avg_cash_incentive FROM ranked;

6. 名单迁移结构

WITH top_old AS ( SELECT creator_id FROM creator_scored QUALIFY ROW_NUMBER() OVER (ORDER BY old_rule_score DESC) <= 4200 ), top_new AS ( SELECT creator_id FROM creator_scored QUALIFY ROW_NUMBER() OVER (ORDER BY priority_score DESC) <= 4200 ) SELECT CASE WHEN o.creator_id IS NOT NULL AND n.creator_id IS NULL THEN '旧名单移出' WHEN o.creator_id IS NULL AND n.creator_id IS NOT NULL THEN '新名单补入' END AS movement_type, c.creator_segment, COUNT(*) AS creator_cnt FROM creator_scored c LEFT JOIN top_old o ON c.creator_id = o.creator_id LEFT JOIN top_new n ON c.creator_id = n.creator_id WHERE (o.creator_id IS NOT NULL AND n.creator_id IS NULL) OR (o.creator_id IS NULL AND n.creator_id IS NOT NULL) GROUP BY 1,2 ORDER BY 1,3 DESC;

7. SRM 与护栏日监控

SELECT dt, experiment_group, COUNT(DISTINCT creator_id) AS creator_cnt, SUM(impression) AS total_impression, SUM(cash_incentive) AS total_incentive, AVG(retention_proxy_7d) AS retention_proxy_7d, AVG(unit_incentive_revenue_7d) AS unit_incentive_revenue_7d, AVG(complaint_flag) AS complaint_rate, AVG(cheat_flag) AS cheat_rate, AVG(safety_flag) AS safety_risk_rate FROM experiment_daily_monitor GROUP BY 1,2 ORDER BY 1,2;

SRM 需要同时看 creator 数、曝光和预算分配，不能只看样本量。

样本说明：数据为合成样本；离线结果仅用于判断是否进入实验，不替代线上因果识别。
