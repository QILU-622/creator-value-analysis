# 创作者资源重配分析：离线诊断、规则回测与实验设计

## 项目定位
这个项目的定位不是“已经完成因果验证的 A/B 实验复盘”，而是：

> **围绕创作者资源配置效率，完成一套从问题诊断、规则设计、同规模回测到实验上线方案的业务分析闭环。**

当前证据支持进入 **creator 级分层灰度实验**，不支持跳过线上验证直接全量切换。

---

## 核心业务问题
在流量和现金激励预算有限时，平台是否把资源持续配置给了更可能形成稳定供给、留存和收入的创作者？

这个问题需要同时回答四层判断：

1. 损耗主要发生在哪个阶段；
2. 资源分配是否仍带有历史曝光惯性；
3. 是否存在值得重新分配资源的关键人群；
4. 离线证据是否足以支持进入灰度实验。

---

## 数据范围
- 创作者样本：**18,742**
- 内容样本：**326,418**
- 周度活跃记录：**299,872**
- 周度激励记录：**299,872**
- 样本性质：**合成样本**

---

## 关键发现
### 1）问题位置清晰
供给漏斗显示，注册到首发并不是主要瓶颈，损耗集中在 **首发后到稳定经营** 阶段：

- 完成首发：**80.6%**
- 连续活跃 ≥ 2 周：**69.7%**
- 连续活跃 ≥ 4 周：**45.7%**
- 已开通变现：**31.6%**
- 稳定经营且已开通变现：**15.0%**

### 2）资源错配成立
- Top 20% 创作者拿走 **42.9%** 曝光，但只贡献 **30.2%** 收入；
- **高曝光低变现** 创作者共 **2,620** 人，占创作者 **14.0%**，却占曝光 **35.0%**；
- **高潜低激励** 创作者共 **1,297** 人，占创作者 **6.9%**，但 30 日留存 **62.4%**、千次曝光收入 **98.1**。

### 3）同规模回测方向成立
在 **Top 4,200** 的同规模比较下：

- 30 日留存：**48.7% → 58.1%（+9.4ppt）**
- 单位激励收入：**69.6 → 78.4（+12.7%）**
- 人均现金激励：**550 → 625（+13.6%）**
- Bootstrap 95% 区间：
  - 留存提升 **8.8–10.1ppt**
  - 单位激励收入提升 **9.6%–16.0%**

### 4）扩量应分阶段推进
不同 Top N 档位下，留存和单位激励收入提升方向均为正，但随着投放规模扩大，边际改善递减，因此更适合 **按池、按垂类、按预算档位** 分层扩量。

## 策略设计
### 1）执行框架
资源分配按三步执行：

1. **准入过滤**：过滤安全、作弊、活跃不足，以及长期低效但资源占用高的人群；
2. **排序加权**：在保留旧规则主体的前提下，引入留存、变现效率、低资源占用三个修正信号；
3. **名单复核**：拟降配名单进入复核队列，避免误伤边界样本。

### 2）首轮灰度口径
```text
新优先级分数
= 0.60 × 原规则分位
+ 0.20 × 留存分位
+ 0.15 × 变现效率分位（缩尾后）
+ 0.05 × 低资源占用分位
```

### 3）为什么选这组权重
这组权重不是宣称“离线最优”，而是首轮灰度下更稳妥的折中：

- 保留 **0.60** 的旧规则权重，控制名单迁移速度；
- 给留存 **0.20** 的显式权重，让长期供给质量进入主排序；
- 给效率 **0.15** 的修正权重，避免短期波动过度主导；
- 给低资源占用 **0.05** 的纠偏权重，只作补充信号，不单独决定名次。

在候选方案里，D 方案的名单重合率、主指标改善和运营承接性更均衡，因此作为首轮灰度方案更合适。

## 解释边界
这个项目最重要的边界不是“有没有提升”，而是：

- **历史回测不是因果识别**；
- 旧规则与历史资源倾斜高度相关；
- 补入名单在历史上并未真实拿到同等资源；
- 线上实验需要验证的，是资源重配带来的**增量效果**，而不是复现离线点估计。

因此，这个项目更适合表述为：

> **离线诊断与规则回测支持进入实验，实验结果才决定是否扩量。**

## 实验设计
| 项目 | 建议 |
|---|---|
| 随机化单位 | creator 级分层随机实验 |
| 实验动作 | 对高潜低激励池做增配；对高曝光低变现池做降配 / 复核 |
| 主指标 | 30 日留存率；单位激励收入 |
| 护栏指标 | 人均现金激励；供给流失率；投诉率；作弊率；内容安全 |
| 建议分层 | 内容垂类；历史曝光档位；创作者体量；是否已开通变现 |

### 暂停或回滚条件
- 连续 3 天出现显著 SRM / 分流漂移；
- 单位激励收入转负且无恢复迹象；
- 投诉率、作弊率、内容安全明显恶化；
- week 1 抬升但 week 4 回落，且 holdout 不支持继续扩量。
